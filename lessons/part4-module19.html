Part 4, Module 19: Capstone Project: Planning & Architecture

Defining game features (player login, matchmaking, actions)

Before you start coding, it's important to plan the features of your game. This includes:

  • Player authentication (login/registration)
  • A matchmaking system to connect players
  • Real-time game actions (e.g., moving a character, casting a spell)
Designing the database schema for users and game state

You'll need to design a database schema to store user data and game state. This might include tables for:

  • users (id, username, password_hash)
  • games (id, status, created_at)
  • game_players (game_id, user_id, score)
Designing the WebSocket message formats for real-time events

You need to define the structure of the messages that will be sent over the WebSocket connection. A common approach is to use a JSON object with a type field to indicate the type of message.

{
  "type": "player_move",
  "payload": {
    "x": 10,
    "y": 20
  }
}
Assignment

Your assignment for this module is to:

  1. Create a document outlining the features of your game.
  2. Design the database schema for your game.
  3. Define the WebSocket message formats for your game.
