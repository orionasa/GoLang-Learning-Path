Introduction to Web Frameworks in Go and why Fiber

While Go's standard net/http package is powerful, web frameworks can provide a more structured and efficient way to build web applications and APIs. Fiber is a web framework for Go that is inspired by Express.js. It's known for its high performance and low memory footprint.

Setting up a new Fiber project: Routing, Handlers, and Context

To start a new Fiber project, you first need to install it:

go get github.com/gofiber/fiber/v2

Here's a basic Fiber application:

package main

import "github.com/gofiber/fiber/v2"

func main() {
    app := fiber.New()

    app.Get("/", func(c *fiber.Ctx) error {
        return c.SendString("Hello, World ðŸ‘‹!")
    })

    app.Listen(":3000")
}
  • Routing: You can define routes using methods like app.Get(), app.Post(), etc.
  • Handlers: Handlers are functions that receive a fiber.Ctx object, which provides access to the request and response.
  • Context: The fiber.Ctx object (context) holds the request and response objects, and allows you to pass data between middleware and handlers.
Handling Requests and Responses (JSON, query params, etc.)

Fiber makes it easy to handle different types of requests and responses.

JSON Response
app.Get("/user", func(c *fiber.Ctx) error {
    return c.JSON(fiber.Map{
        "name": "John",
        "age":  30,
    })
})
Query Parameters
app.Get("/search", func(c *fiber.Ctx) error {
    query := c.Query("q")
    return c.SendString("Search query: " + query)
})
