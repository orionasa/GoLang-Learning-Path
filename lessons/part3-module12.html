Using Middleware in Fiber (Logging, Recovery, CORS)

Middleware are functions that execute before your route handlers. They are useful for tasks like logging, authentication, and error handling.

Fiber comes with a set of built-in middleware.

package main

import (
    "log"
    "github.com/gofiber/fiber/v2"
    "github.com/gofiber/fiber/v2/middleware/logger"
    "github.com/gofiber/fiber/v2/middleware/recover"
)

func main() {
    app := fiber.New()

    app.Use(logger.New())
    app.Use(recover.New())

    // ... your routes
}
Introduction to the `database/sql` package

The database/sql package provides a generic SQL interface. You use it with a database driver to connect to a specific database.

Setting up PostgreSQL (or MySQL) and connecting from Go

To connect to a database, you need a database driver. For PostgreSQL, you can use pq or pgx.

go get github.com/lib/pq

import (
    "database/sql"
    _ "github.com/lib/pq"
)

func main() {
    connStr := "user=pqgotest dbname=pqgotest sslmode=verify-full"
    db, err := sql.Open("postgres", connStr)
    if err != nil {
        log.Fatal(err)
    }
    defer db.Close()
}
