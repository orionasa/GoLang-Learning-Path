Part 1, Module 3: Collections and Functions

Arrays, Slices, and their relationship

Arrays have a fixed size and a specific type.

var a [2]string
a[0] = "Hello"
a[1] = "World"

Slices are a more flexible, powerful interface to arrays. They are dynamically-sized and are more common in Go.

primes := []int{2, 3, 5, 7, 11, 13}
Maps for key-value storage

Maps are Go's built-in associative data type (sometimes called hash maps or dicts in other languages).

m := make(map[string]int)
m["k1"] = 7
m["k2"] = 13
fmt.Println("map:", m)
Defining and Calling Functions

Functions are central in Go. They can be defined with the func keyword.

func add(a int, b int) int {
    return a + b
}

func main() {
    res := add(42, 13)
    fmt.Println("42 + 13 =", res)
}
Multiple Return Values and Variadic Functions

Go functions can return multiple values. This is often used for returning a result and an error value.

func vals() (int, int) {
    return 3, 7
}

func main() {
    a, b := vals()
    fmt.Println(a)
    fmt.Println(b)
}

Variadic functions can be called with any number of trailing arguments.

func sum(nums ...int) {
    fmt.Print(nums, " ")
    total := 0
    for _, num := range nums {
        total += num
    }
    fmt.Println(total)
}

func main() {
    sum(1, 2)
    sum(1, 2, 3)
}
The `error` type for basic error handling

Go uses an explicit, value-based error handling mechanism. Functions that can fail return an error as their last return value.

func Sqrt(x float64) (float64, error) {
	if x < 0 {
		return 0, errors.New("cannot Sqrt negative number")
	}
	return math.Sqrt(x), nil
}
