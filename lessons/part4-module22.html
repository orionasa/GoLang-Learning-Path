Part 4, Module 22: Capstone: Containerization & Deployment

Writing a `Dockerfile` to containerize the application

A Dockerfile is a text document that contains all the commands a user could call on the command line to assemble an image. Docker can build images automatically by reading the instructions from a Dockerfile.

FROM golang:1.19-alpine

WORKDIR /app

COPY go.mod ./
COPY go.sum ./
RUN go mod download

COPY . .

RUN go build -o /server

EXPOSE 8080

CMD [ "/server" ]
Conceptual overview of deploying to a cloud provider (e.g., AWS, DigitalOcean)

Once you have containerized your application, you can deploy it to a cloud provider like AWS, Google Cloud, or DigitalOcean. This typically involves:

  • Pushing your Docker image to a container registry (e.g., Docker Hub, AWS ECR).
  • Running your container on a cloud service (e.g., AWS ECS, Google Kubernetes Engine, DigitalOcean App Platform).
Final testing and project wrap-up

Before you deploy your application to production, it's important to thoroughly test it to ensure that it's working as expected.
