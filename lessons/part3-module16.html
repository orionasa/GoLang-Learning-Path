Part 3, Module 16: Caching for Performance

Caching Strategies: In-memory vs. Distributed

Caching is a technique used to store frequently accessed data in a temporary storage location (cache) to reduce the latency and load on the primary data source.

  • In-memory caching: Storing data in the application's memory. It's fast but not persistent and not suitable for distributed systems.
  • Distributed caching: Using an external service like Redis or Memcached to store cached data. This is suitable for distributed systems and provides persistence.
Integrating Redis for distributed caching and state

Redis is a popular in-memory data store that can be used for caching, session management, and more. You can use a library like go-redis to interact with Redis from your Go application.

go get github.com/go-redis/redis/v8

rdb := redis.NewClient(&redis.Options{
    Addr:     "localhost:6379",
    Password: "", // no password set
    DB:       0,  // use default DB
})
