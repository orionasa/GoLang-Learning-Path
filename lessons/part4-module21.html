Part 4, Module 21: Capstone: Real-Time Game Logic

Implement a matchmaking queue using Redis lists or sets

You can use Redis to implement a simple matchmaking queue. When a player wants to join a game, you can add them to a Redis list or set. When there are enough players in the queue, you can create a new game and add the players to it.

Create a real-time game session manager using WebSockets

You will need to create a game session manager to handle the real-time game logic. This will involve:

  • Managing the state of each game.
  • Handling WebSocket connections for each player in the game.
  • Broadcasting game state updates to all players in the game.
Handle authoritative game logic on the server (e.g., player moves)

For security and to prevent cheating, all game logic should be handled on the server. The client should only send user input to the server, and the server should be responsible for updating the game state and broadcasting it to the clients.

Assignment

Your assignment for this module is to:

  1. Implement a matchmaking queue using Redis.
  2. Create a game session manager to handle the real-time game logic.
  3. Implement a basic game loop that updates the game state and broadcasts it to the clients.
