Go Modules: Managing dependencies (`go.mod`, `go.sum`)

Go modules are the standard way to manage dependencies in Go. A module is a collection of Go packages that are released together.

  • go mod init <module-path>: Initializes a new module.
  • go.mod: Defines the module's path and its dependencies.
  • go.sum: Contains the checksums of the module's dependencies.
Writing Unit Tests with the `testing` package

Go has a built-in testing framework in the testing package. Tests are written in files that end with _test.go.

func TestAbs(t *testing.T) {
    got := Abs(-1)
    if got != 1 {
        t.Errorf("Abs(-1) = %f; want 1", got)
    }
}

You can run tests using the go test command.

Benchmarking your code for performance analysis

The testing package also supports benchmarking. Benchmark functions are similar to test functions, but they are run multiple times to measure performance.

func BenchmarkFib(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Fib(10)
    }
}

You can run benchmarks using the go test -bench=. command.
