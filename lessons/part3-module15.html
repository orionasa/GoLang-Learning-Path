Part 3, Module 15: Advanced Security Practices

Input Validation and Sanitization to prevent attacks

You should always validate and sanitize user input to prevent attacks like SQL injection and Cross-Site Scripting (XSS).

Libraries like go-validator can help with this.

Managing secrets with Environment Variables

Never hardcode secrets like API keys and database passwords in your code. Use environment variables to store them.

Libraries like godotenv can help you load environment variables from a .env file during development.

Implementing Rate Limiting and basic DoS protection

Rate limiting can help protect your API from abuse and Denial-of-Service (DoS) attacks. Fiber has a built-in rate limiting middleware.

app.Use(limiter.New(limiter.Config{
    Max: 100,
    Expiration: 1 * time.Minute,
}))
