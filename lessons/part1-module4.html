Part 1, Module 4: Structs, Methods & Interfaces

Composite Types: Structs and Pointers

A struct is a collection of fields. It's useful for grouping data together to form records.

type Vertex struct {
    X int
    Y int
}

func main() {
    v := Vertex{1, 2}
    fmt.Println(v.X)
}

A pointer holds the memory address of a value. The & operator generates a pointer to its operand.

p := &v
p.X = 1e9
fmt.Println(v)
Defining Methods on Structs

Go does not have classes. However, you can define methods on types. A method is a function with a special receiver argument.

type Vertex struct {
    X, Y float64
}

func (v Vertex) Abs() float64 {
    return math.Sqrt(v.X*v.X + v.Y*v.Y)
}

func main() {
    v := Vertex{3, 4}
    fmt.Println(v.Abs())
}

You can declare methods with pointer receivers to modify the value to which the receiver points.

Interfaces: Defining and Implementing Behavior

An interface type is defined as a set of method signatures. A value of interface type can hold any value that implements those methods.

type Abser interface {
    Abs() float64
}
The Empty Interface (`interface{}`) and Type Assertions

The interface type that specifies zero methods is known as the empty interface. An empty interface may hold values of any type.

var i interface{}
i = 42
i = "hello"

A type assertion provides access to an interface value's underlying concrete value.

s := i.(string)
Go's philosophy: Composition over Inheritance

Go does not support inheritance. Instead, it encourages composition through embedding structs and interfaces.
