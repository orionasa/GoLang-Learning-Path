Part 2, Module 5: Introduction to Concurrency

Understanding Concurrency vs. Parallelism

Concurrency is about dealing with lots of things at once. Parallelism is about doing lots of things at once.

Concurrency is a way to structure your program to handle multiple tasks, while parallelism is the simultaneous execution of those tasks.

Goroutines: The lightweight threads of Go

A goroutine is a lightweight thread managed by the Go runtime. You can start a new goroutine by using the go keyword.

func say(s string) {
    for i := 0; i < 5; i++ {
        time.Sleep(100 * time.Millisecond)
        fmt.Println(s)
    }
}

func main() {
    go say("world")
    say("hello")
}
Channels: The primary way to communicate between Goroutines

Channels are the pipes that connect concurrent goroutines. You can send values into channels from one goroutine and receive those values into another goroutine.

ch := make(chan int)
ch <- 1 // Send a value to the channel
x := <-ch // Receive a value from the channel
Unbuffered vs. Buffered Channels

Unbuffered channels require both the sender and receiver to be ready. Buffered channels have a capacity and can store a limited number of values without a corresponding receiver.

ch := make(chan int, 2) // Buffered channel with a capacity of 2
ch <- 1
ch <- 2
